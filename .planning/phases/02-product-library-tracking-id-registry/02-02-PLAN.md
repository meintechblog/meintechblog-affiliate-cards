/**
 * Saves a product to the library. ASIN-only at this stage;
 * title/images can be enriched later via Amazon Creators API.
 */
private function save_product(string $asin): void {
    if ($this->productLibrary === null) {
        return;
    }
    $this->productLibrary->insert([
        'asin'       => $asin,
        'title'      => '',
        'detail_url' => "https://www.amazon.de/dp/{$asin}",
        'image_url'  => '',
    ]);
}
```

<?php
/**

 * Or:    cd /path/to/wordpress && php -r "define('ABSPATH',__DIR__.'/'); require 'wp-load.php';" && php wp-content/plugins/meintechblog-affiliate-cards/scripts/backfill-tracking-ids.php

    // Allow direct execution from plugin directory if WP is already loaded

    if (file_exists($wp_load)) {
        require_once $wp_load;
    } else {
        echo "Error: WordPress not found. Run via: wp eval-file scripts/backfill-tracking-ids.php\n";
        exit(1);
    }
}
```

2. Read the backfill file:
```php
$backfill_file = __DIR__ . '/../.planning/data/tracking-ids-backfill.txt';
if (!file_exists($backfill_file)) {
    echo "Error: Backfill file not found at {$backfill_file}\n";
    exit(1);
}
$lines = file($backfill_file, FILE_IGNORE_NEW_LINES | FILE_SKIP_EMPTY_LINES);
```

3. Create the tracking registry instance and ensure table exists:
```php
$registry = new MTB_Affiliate_Tracking_Registry();
if (MTB_Affiliate_Tracking_Registry::needs_upgrade()) {
    MTB_Affiliate_Tracking_Registry::create_table();
}
```

4. Import each line, tracking counts:
```php
$imported = 0;
$skipped = 0;
$total = count($lines);

foreach ($lines as $line) {
    $trackingId = trim($line);
    if ($trackingId === '') {
        continue;
    }
    if ($registry->register($trackingId)) {
        $imported++;
    } else {
        $skipped++;
    }
}

echo "Backfill complete: {$imported} imported, {$skipped} skipped (already exist or invalid), {$total} total lines.\n";
```

The `register()` method already handles duplicates (returns false if exists), so re-running the script is safe/idempotent.

Note: The backfill file contains some known anomalies (e.g., `eintechblog-231104-21` missing 'm', `meintechblog-2009011-21` with 7-digit date, `facebook-2017-21`, `klimahofmann-2015-21`, `wwwmeintechbl-21`). These should be imported as-is -- they are real historical tracking IDs that were actually used. The script does NOT validate format; it imports exactly what is in the file.
