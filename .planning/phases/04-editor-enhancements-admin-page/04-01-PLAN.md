  <action>
Inside the AffiliateCardsEdit function body, immediately after the existing hook declarations (after the `const hydrationAsinRef = useRef(...)` line, before the `function updateItem` declaration), add two new state variables and one new useEffect:

```javascript
const [ products, setProducts ] = element.useState( [] );
const [ productsLoaded, setProductsLoaded ] = element.useState( false );

useEffect( function () {
    if ( productsLoaded ) { return; }
    var root = ( window.wpApiSettings && window.wpApiSettings.root )
        ? window.wpApiSettings.root.replace( /\/+$/, '' )
        : '/wp-json';
    var headers = {};
    if ( window.wpApiSettings && window.wpApiSettings.nonce ) {
        headers[ 'X-WP-Nonce' ] = window.wpApiSettings.nonce;
    }
    window.fetch( root + '/' + PRODUCTS_ENDPOINT + '?limit=50', {
        credentials: 'same-origin',
        headers: headers
    } )
        .then( function ( r ) { return r.ok ? r.json() : []; } )
        .then( function ( data ) {
            setProducts( Array.isArray( data ) ? data : [] );
            setProductsLoaded( true );
        } )
        .catch( function () { setProductsLoaded( true ); } );
}, [] );
```

CRITICAL: The dependency array MUST be `[]` (empty). Using `[currentAsin]` would re-fetch on every ASIN change and cause an infinite loop. The `productsLoaded` guard prevents double-fetching on StrictMode double-invocations.

Do NOT add a `useState` import — `element.useState` is already destructured at the top of the IIFE via `const useEffect = element.useEffect`. Use `element.useState` directly (same pattern as the existing `useRef` which uses `element.useRef`).

Insert between #1 and #2:

```javascript
products.length > 0 && el( SelectControl, {
    label: i18n.__( 'Aus Bibliothek wählen', 'meintechblog-affiliate-cards' ),
    value: '',
    options: [ { label: '— Produkt auswählen —', value: '' } ].concat(
        products.map( function ( p ) {
            return {
                label: ( p.title || p.asin ) + ' (' + p.asin + ')',
                value: p.asin
            };
        } )
    ),
    onChange: function ( value ) {
        if ( value ) {
            updateItem( 'asin', value.trim().toUpperCase() );
        }
    }
} ),
```

CRITICAL: `value` must always be `''` (never bound to `item.asin`). This makes the dropdown a write-only action picker. If `value` were bound to `currentAsin`, selecting a product would set the dropdown's displayed selection, and subsequent renders could re-trigger hydration via the ASIN useEffect in a loop. The user sees the chosen ASIN in the TextControl above, not in the dropdown.

The `products.length > 0 &&` guard hides the dropdown when the product library is empty or not yet loaded, avoiding a confusing empty select element.

Do NOT add a new `SelectControl` import — it is already declared at the top of the IIFE: `const SelectControl = components.SelectControl;`
